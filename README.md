# 県市切替マクロ

施工計画書・施工体制台帳ブックで、D3 の値に合わせて県用・市用のシート表示を切り替えるためのマクロです。

この版では、VBEで文字化けしにくいように、マクロ本文を **ASCII安全版** にしています。シート名やナビ表示は日本語のまま動きます。

---

## まず覚えること

| D3の状態 | 動き |
| --- | --- |
| `県` | 県土木向けシートを表示し、市役所向けシートを非表示にします。 |
| `市` | 市役所向けシートを表示し、県土木向けシートを非表示にします。 |
| 空欄 | 県・市どちらのシートも表示します。 |

提出先がまだ決まっていないとき、または両方の帳票を見たいときは、D3を空欄にしてください。

---

## 重要：前に文字化けした人向け

VBE画面で赤い文字や意味不明な記号だらけになっている場合は、古いモジュールが壊れています。

新しいマクロを入れる前に、VBE左側のプロジェクトから、古い文字化けモジュールを削除してください。

目安として、次のような名前のモジュールが古いものです。

```text
県市切替
```

削除手順：

1. `Alt + F11` で VBE を開く
2. 左側のプロジェクトで `標準モジュール` を開く
3. 文字化けしているモジュール、または古い `県市切替` モジュールを右クリック
4. `○○ の解放` を選ぶ
5. エクスポートするか聞かれたら、不要なら `いいえ`
6. その後、新しい `県市切替マクロ.bas` をインポートする

新しいモジュール名は、VBE上では **`KenShiSwitch`** と表示されます。

## 導入手順

### 1. マクロを取り込む

1. 対象のExcelブックを開く
2. `Alt + F11` で VBE を開く
3. メニュー `ファイル → ファイルのインポート` を選ぶ
4. `県市切替マクロ.bas` を選ぶ
5. 左側に `KenShiSwitch` モジュールが追加されたことを確認する

### 2. 初期設定を実行する

1. Excelに戻る
2. `Alt + F8` を押す
3. `Setup_KenShi` を選んで実行する
4. 完了メッセージが出ればOK

これでD3にドロップダウンが付き、`県` / `市` / 空欄で切り替えられるようになります。

---

## 使い方

1. `施工計画書（はじめに記入）` シートを開く
2. D3 の ▼ から `県` または `市` を選ぶ
3. 県用・市用のシート表示が自動で切り替わる
4. D3を空欄にすると、県・市どちらのシートも表示される
5. H3:K30 に「提出書類ナビ」が表示される

---

## 提出書類ナビ

入力シートの **H3:K30** に、対象シートへ移動するためのリンク一覧が自動で表示されます。

| D3 | ナビの表示 |
| --- | --- |
| `県` | 県土木向けシートへのリンク |
| `市` | 市役所向けシートへのリンク |
| 空欄 | `【県土木】` と `【市役所】` の両方のリンク |

### 注意

**H3:K30 はナビ専用領域です。**

この範囲に手書きメモや別のデータを置くと、D3切替時に上書きされます。

---

## 提出前・配布前チェック

配布前、または友人に渡す前に、次のチェックをしてください。

1. `Alt + F8`
2. `Check_KenShiSettings` を選ぶ
3. 実行する
4. `Check completed. No problems were found.` と出ればOK

このチェックでは、次を確認します。

- 入力シートが存在するか
- D3の値が `県` / `市` / 空欄 のどれかになっているか
- 県用シート一覧のシートが存在するか
- 市用シート一覧のシートが存在するか
- 県用・市用で重複したシート名がないか
- D3変更時に動く `Worksheet_Change` が入っているか

---

## トラブル対応

### D3を変えてもシートが切り替わらない

`Worksheet_Change` が入っていない可能性があります。

1. `Alt + F8`
2. `Setup_KenShi` を再実行
3. `Check_KenShiSettings` を実行

それでもだめな場合は、VBEのシートモジュールに手動で次を貼ります。

```vb
Private Sub Worksheet_Change(ByVal Target As Range)
    If Intersect(Target, Me.Range("D3")) Is Nothing Then Exit Sub
    If Target.Cells.Count > 1 Then Exit Sub
    KenShiSheetSwitch CStr(Target.Value)
End Sub
```

### 文字化けした赤いコードが出る

古い `.bas` が文字コード違いで壊れて取り込まれています。

対処：

1. 文字化けしている古い標準モジュールを削除する
2. 新しい `県市切替マクロ.bas` をインポートする
3. `Setup_KenShi` を実行する

新しい版は、コード本文をASCII安全版にしているため、VBEで文字化けしにくくなっています。

### 「Input sheet was not found」と出る

入力シート名が違う可能性があります。

想定しているシート名は次です。

```text
施工計画書（はじめに記入）
```

シート名を合わせるか、マクロ内の `InputSheetName()` を実際のシート名に合わせて修正してください。

## カスタマイズする場合

### 入力セルを変える

マクロ冒頭のここを変更します。

```vb
Private Const INPUT_CELL As String = "D3"
```

### ナビの場所を変える

```vb
Private Const NAV_RANGE As String = "H3:K30"
```

### 対象シートを変える

`KenSheetNames()` と `ShiSheetNames()` の中身を変更します。

ただし、この版では文字化け防止のため、日本語シート名を `U(&H....)` 形式で書いています。通常は自分で触らない方が安全です。
